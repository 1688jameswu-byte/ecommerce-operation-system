# ROADMAP.md

## 1. 项目建设总方向

本项目是公司内部电商运营中心，不是一次性建设完整ERP。

建设原则：

* 先解决真实业务问题。
* 先让核心流程跑通，再逐步增强。
* 先做轻量闭环，再做复杂自动化。
* 先稳定旧系统，再扩展新模块。
* 新业务优先使用更合理的数据结构。
* 不为了技术升级而破坏现有业务稳定性。

当前总方向：

```text
旧模块稳定运行
1688新业务数据库化试点
运营闭环逐步完善
薪资绩效继续补齐
后续再考虑全系统数据库迁移
```


## 3. 近期优先级

### 第一优先级：系统稳定和边界确认

目标：

* 保持现有系统稳定
* 明确菜单结构
* 明确数据存储边界
* 明确文档读取规则
* 避免 Codex 每次任务读太多文档

重点任务：

* 建立 AGENTS.md
* 建立 PROJECT.md
* 建立 ROADMAP.md
* 1688业务作为一级菜单
* 1688业务放在“运营闭环”和“运营工具”之间
* AI图片提示词中心放在“运营工具”下面

---

### 第二优先级：1688业务第一阶段

目标：

建立 1688 业务基础框架。

1688业务当前定位：

```text
半自动上架资料库 + 人工上架任务闭环
```

不做：

* 1688 API全自动上架
* 浏览器模拟自动发布
* 爬虫式自动发布

第一阶段二级菜单：

```text
产品库
上架任务
图片素材
供应商管理
1688设置
```

第一阶段数据库：

```text
1688业务直接使用 PostgreSQL
旧模块继续 JSON
```

第一阶段核心表：

```text
1688_products
1688_product_skus
1688_product_images
1688_suppliers
1688_listing_tasks
1688_stores
1688_settings
```

第一阶段目标：

* 产品能录入
* SKU能在产品详情中管理
* 图片能关联产品
* 供应商能维护
* 产品资料能检查完整性
* 可生成上架任务
* 人工上架后可回填1688链接

---

### 第三优先级：1688业务第二阶段

目标：

让1688业务从“能录入”升级为“能高效上架”。

重点任务：

* 上架资料包
* 一键复制标题
* 一键复制关键词
* 一键复制卖点
* 一键复制详情文案
* 一键复制SKU表格
* 上架任务状态流转
* 上架失败原因记录
* 业务员上架数量统计
* 产品资料缺失提醒
* 图片状态管理

此阶段仍然不做全自动上架。

---

### 第四优先级：权限和字段级控制

目标：

让不同角色看到不同内容。

重点任务：

* 1688业务员只能看自己负责的产品、店铺、任务
* 管理员能看全部
* 主管能看所属范围
* 普通业务员不显示拿货价
* 普通业务员不显示供应商核心成本
* 后端 API 增加权限校验
* 前端隐藏无权限菜单和按钮

注意：

不能只靠前端隐藏，后端也要校验。

---

### 第五优先级：后台体验和性能优化

目标：

让后台适合长期办公。

重点任务：

* 后台页面浅色办公风格
* 表格、表单、弹窗统一优化
* 大数据页面分页
* 大数据页面懒加载
* 大数据页面虚拟滚动
* 编辑使用弹窗或抽屉
* 删除、停用、覆盖导入强确认
* 删除按钮与上传按钮保持距离

重点关注页面：

* 流量转化导入
* 异常结果中心
* 打卡记录
* 薪资绩效表格页
* 1688产品库
* 1688图片素材

---

### 第六优先级：薪资绩效系统继续完善

目标：

让薪资绩效系统稳定可用。

重点任务：

* 员工档案导入
* 打卡记录按月搜索
* 打卡记录分页或虚拟滚动
* 考勤规则配置
* 工资周期
* 计时工资
* 计件工资
* 绩效工资
* 工资统计

注意：

* 员工匹配以姓名严格匹配为主。
* 不建议做员工别名机制。
* 重新导入同一周期数据时，允许更新历史 unmatched_employee 记录。
* 薪资旧数据继续 JSON，不因1688 PostgreSQL接入而迁移。

---

### 第七优先级：TEMU运营闭环继续完善

目标：

让TEMU从“看数据”升级为“发现问题、处理问题、复盘问题”。

重点任务：

* 异常结果中心
* 运营任务中心
* 异常转任务
* 任务处理说明
* 任务状态流转
* 任务复盘
* 避免同一异常重复生成任务
* 有效上新录入
* 首单商品数排名
* 运营销售额排名
* 店铺销售额排名

注意：

TEMU旧数据继续 JSON，不在当前阶段迁移 PostgreSQL。

* 1688爬虫式上架

## 7. 阶段判断标准

### 1688第一阶段完成标准

满足以下条件即可认为第一阶段完成：

* 1688业务为一级菜单
* 二级菜单结构正确
* PostgreSQL连接正常
* 产品库可新增、编辑、查看
* 产品详情可管理SKU
* 图片素材可关联产品
* 供应商可维护
* 可生成上架任务
* 可回填1688商品链接
* 不影响旧模块
* npm run build 通过

---

### 1688第二阶段完成标准

满足以下条件即可认为第二阶段完成：

* 产品详情页有上架资料包
* 支持复制标题、卖点、详情文案、SKU表格
* 支持资料完整性检查
* 缺失资料有明确提示
* 上架任务状态流转清晰
* 已上架产品可回填链接
* 业务员能看到自己的上架任务

---

### 权限阶段完成标准

满足以下条件即可认为权限阶段完成：

* 菜单按账号权限展示
* 店铺按账号权限展示
* 任务按负责人或管理范围展示
* 普通业务员看不到拿货价和核心成本
* 后端 API 有权限校验
* 无权限访问有明确提示

---

## 8. 开发节奏原则

每个阶段都要小步推进。

推荐方式：

```text
先做数据结构
再做 API
再做页面列表
再做新增编辑
再做详情页
再做状态流转
最后做权限和体验优化
```

每次任务尽量只解决一个明确问题。

不要把多个大模块混在一次任务里。

---

## 9. 总结

当前路线：

```text
先稳定旧系统
再建设1688业务
1688直接使用PostgreSQL
旧模块暂时继续JSON
先做半自动上架
不做全自动API上架
先跑通产品库、图片、供应商、任务闭环
再考虑询盘、客户、订单、数据分析
```
