# AI 反馈样本使用说明

本说明用于人工评估从任务中心导出的 AI 复盘反馈 JSON。当前阶段不自动训练、不自动改 prompt。

## 反馈样本来源

任务中心支持两种复制方式：

- 单条任务：点击 AI 任务卡片里的 `复制 AI 反馈`
- 当前筛选结果：点击任务工作台右上角 `复制当前 AI 反馈`

导出的 JSON 主要包含：

- `sourceContent`：AI 任务生成时的问题背景
- `suggestion`：AI 建议内容
- `resultNote`：运营实际处理结果
- `reviewStatus`：复盘结论
- `reviewNote`：复盘备注

## 人工评估口径

建议按以下字段整理样本：

| 字段 | 判断口径 |
| --- | --- |
| 建议是否有效 | AI 建议是否帮助运营定位问题或推动处理 |
| 指标是否改善 | 处理后核心指标是否有明确改善 |
| 建议是否可执行 | 动作是否具体、可分派、可验证 |
| 是否误判 | AI 是否把问题归因到错误方向 |
| 缺少什么上下文 | 是否缺商品、广告、活动、库存、价格、售后等数据 |
| Prompt 改进建议 | 下次希望 AI 多问什么、少说什么、输出什么格式 |

## 建议分组

人工整理时可先分为四类：

1. `有效建议`：建议方向正确，处理后有改善。
2. `部分有效`：建议方向有帮助，但需要更多上下文才能执行。
3. `无效建议`：建议方向错误或泛泛而谈。
4. `数据不足`：不是模型问题，主要缺少关键上下文。

## Prompt 改进原则

- 不用单条样本直接改 prompt。
- 至少收集 5-10 条同类问题后再总结共性。
- 优先补充输入上下文，其次再改输出格式。
- 如果问题来自数据缺失，应改 Context 构造，不应只改提示词。
- 如果建议太泛，应要求 AI 输出可验证动作、责任角色和观察指标。

## 后续可演进

后续可以新增“AI 反馈样本池”，但需要先设计数据结构。当前阶段只建议人工复制 JSON 到文档或表格中评估。
