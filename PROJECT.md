# 电商运营中心项目说明

## 1. 项目定位
本项目是公司内部电商运营管理系统，用于统一管理运营数据、任务闭环、1688业务、薪资绩效及运营工具。  
核心模块：
- TEMU运营数据中心
- 运营异常分析与任务闭环
- 1688业务管理
- AI图片提示词中心
- 薪资绩效系统
- 后台账号权限管理

设计原则：
- 分阶段迭代，先解决真实业务问题
- 后台管理页面浅色、易用，运营大屏深色可保留
- 危险操作二次确认
- 新功能兼容旧数据

---

## 2. 技术与数据存储
- 技术栈：React + TypeScript + Vite
- 旧模块仍使用 JSON 文件存储（TEMU、薪资、运营闭环等）
- 新模块1688业务直接使用 PostgreSQL
- 前端通过后端 API 访问 PostgreSQL
- 数据库连接信息使用环境变量，不允许写死账号密码
- 旧模块暂不迁移

## 4. 1688业务核心定位
- 半自动上架资料库 + 人工上架任务闭环
- 不做全自动API上架或浏览器模拟发布
- 系统负责：
  - 产品资料管理、SKU规格、图片素材、供应商管理
  - 生成上架资料包
  - 上架完整性检查
  - 生成上架任务
  - 人工上架完成后回填1688链接
- 人工负责：
  - 登录1688后台
  - 复制系统资料、上传图片、发布商品
  - 回填商品链接

---

## 5. 产品库设计原则
- 单一产品库页面，SKU在产品详情页表格管理
- 支持：
  - 产品基础信息
  - SKU规格信息
  - 价格信息
  - 图片资料
  - 标题/关键词/卖点/详情文案
  - 上架状态、1688商品链接
  - 生成上架任务、检查完整性
- 产品状态：

资料待完善
待拍照
待修图
待生成文案
待上架
可上架
人工上架中
已上架
上架失败
已下架


---

## 6. 1688半自动上架流程

新增产品
↓
补充SKU规格、价格、图片、供应商、详情文案
↓
系统检查资料完整性
↓
状态变为可上架
↓
生成上架任务
↓
业务员人工登录1688后台发布
↓
回填商品链接
↓
任务状态更新为已上架


---

## 7. 1688 PostgreSQL 数据库方案
- 1688新模块直接使用 PostgreSQL
- 前端不直接访问数据库，全部通过后端 API
- 核心表：
  - 1688_products
  - 1688_product_skus
  - 1688_product_images
  - 1688_suppliers
  - 1688_listing_tasks
  - 1688_stores
  - 1688_settings
- 详细字段设计放入 `docs/1688-business.md`

---

## 8. 权限原则
- 管理员：查看全部模块和数据
- 主管：查看所属业务范围
- 业务员：只看自己负责店铺、产品、任务
- 供应商成本敏感字段需字段级权限
- 危险操作：编辑直接执行，停用二次确认，删除强确认

---

## 9. 页面体验原则
- 后台管理页面浅色主题，表格、表单、弹窗清晰
- 大数据页面需分页/虚拟滚动/懒加载
- 编辑操作优先弹窗/抽屉
- 删除按钮离上传按钮适当间隔
- 运营大屏可保留深色
